## 👋 Welcome

Selamat datang di repository pembelajaran saya.

Repository ini mendokumentasikan perjalanan belajar saya menuju karier sebagai **Cloud Engineer**, **DevOps Engineer**, dan **Cloud Security Engineer** melalui pendekatan yang terstruktur, berorientasi praktik, dan menyerupai implementasi di lingkungan enterprise.

Seluruh materi dipelajari secara bertahap mulai dari **Linux Foundation**, **Cloud Computing**, **Infrastructure as Code**, **Containerization**, hingga **Cloud Native Technologies**, dengan fokus utama pada **70% praktik** dan **30% teori**.

Berbeda dengan repository belajar pada umumnya yang hanya berisi kumpulan command, repository ini menggabungkan:

- 📚 Hands-on Lab
- 🎯 Challenge Lab
- 🏗️ Mini Project
- 📖 Technical Documentation
- 🔍 Troubleshooting & Root Cause Analysis (RCA)
- 🏢 Enterprise Case Study

# 🎯 Learning Goals

Repository ini dibuat sebagai dokumentasi perjalanan belajar saya untuk membangun kompetensi sebagai **Cloud Engineer**, **DevOps Engineer**, dan **Cloud Security Engineer** melalui pendekatan yang terstruktur, berbasis praktik, dan mengikuti alur implementasi yang umum digunakan di lingkungan industri.

Seluruh materi dipelajari secara bertahap, dimulai dari fondasi Linux hingga teknologi Cloud Native, dengan tujuan tidak hanya memahami konsep, tetapi juga mampu mengimplementasikannya pada studi kasus yang menyerupai lingkungan production.

Target utama dari repository ini adalah:

- Menguasai administrasi sistem Linux sebagai fondasi infrastruktur modern.
- Memahami konsep networking yang digunakan pada lingkungan cloud dan DevOps.
- Membangun kemampuan dalam mengelola layanan cloud menggunakan Amazon Web Services (AWS).
- Menguasai Infrastructure as Code (IaC) menggunakan Terraform.
- Memahami containerization menggunakan Docker.
- Mengelola container orchestration menggunakan Kubernetes.
- Membangun dan mengotomasi deployment menggunakan CI/CD Pipeline.
- Menerapkan praktik observability melalui monitoring, logging, dan alerting.
- Memahami prinsip DevSecOps serta implementasi keamanan pada infrastruktur cloud.
- Membangun portfolio teknis yang terdokumentasi secara profesional melalui Hands-on Lab, Challenge Lab, dan Enterprise Mini Project.

# 📚 Learning Approach

Setiap materi pada repository ini dipelajari menggunakan pendekatan bertahap agar tidak hanya memahami teori, tetapi juga mampu menerapkannya pada situasi nyata.

Untuk setiap topik pembelajaran, saya menyusun dokumentasi dalam tiga tahapan utama:

### 📖 Hands-on Lab

Mempelajari konsep dasar, penggunaan command, serta implementasi fitur secara langsung pada server Ubuntu yang berjalan di Amazon EC2.

### 🎯 Challenge Lab

Menyelesaikan berbagai skenario troubleshooting dan investigasi yang menyerupai tiket pekerjaan seorang Junior Linux Administrator, Cloud Engineer, maupun DevOps Engineer.

### 🏗️ Enterprise Mini Project

Mengintegrasikan seluruh materi yang telah dipelajari menjadi sebuah deliverable nyata, seperti audit server, implementasi layanan, atau solusi infrastruktur sederhana yang didokumentasikan secara profesional sebagai bagian dari portfolio GitHub.

---

> **Goal utama repository ini bukan hanya menyelesaikan proses belajar, tetapi membangun kompetensi yang dapat diterapkan pada lingkungan kerja nyata serta menghasilkan portfolio teknis yang menunjukkan kemampuan problem solving, dokumentasi, dan implementasi infrastruktur modern.**

# 🧠 Learning Philosophy

Saya percaya bahwa kemampuan teknis tidak dibangun hanya dengan membaca teori atau menghafal command.

Seorang Cloud Engineer, DevOps Engineer, maupun Linux System Administrator dituntut untuk mampu memahami konsep, menyelesaikan masalah nyata, serta menghasilkan solusi yang dapat dipertanggungjawabkan melalui dokumentasi yang baik.

Oleh karena itu, seluruh materi pada repository ini dipelajari menggunakan pendekatan **70% praktik** dan **30% teori**, dengan setiap topik disusun secara bertahap mulai dari konsep dasar hingga implementasi yang menyerupai lingkungan production.

Repository ini tidak hanya berisi kumpulan command Linux, tetapi juga mendokumentasikan proses berpikir, analisis, troubleshooting, serta penyusunan solusi terhadap berbagai studi kasus yang umum ditemui di dunia kerja.

# 📖 Hands-on Lab

Hands-on Lab merupakan tahap pertama dalam setiap materi pembelajaran.

Pada tahap ini saya mempelajari:

- Konsep dasar suatu teknologi.
- Fungsi setiap command yang digunakan.
- Cara kerja fitur yang tersedia.
- Best Practice penggunaan command.
- Eksperimen langsung pada Ubuntu Server di AWS EC2.

Fokus utama Hands-on Lab adalah membangun pemahaman terhadap **bagaimana suatu teknologi bekerja** sebelum digunakan untuk menyelesaikan permasalahan yang lebih kompleks.

---

# 🎯 Challenge Lab

Setelah memahami dasar-dasar suatu teknologi, pembelajaran dilanjutkan ke Challenge Lab.

Challenge Lab disusun menggunakan pendekatan **Enterprise Ticket Simulation**, yaitu simulasi investigasi yang menyerupai pekerjaan seorang Junior Linux Administrator, Cloud Engineer, maupun DevOps Engineer.

Setiap challenge dirancang untuk melatih kemampuan:

- Investigasi sistem.
- Analisis log.
- Troubleshooting.
- Root Cause Analysis (RCA).
- Pengambilan keputusan berdasarkan data.
- Dokumentasi hasil investigasi.

Pada tahap ini saya tidak hanya menjalankan command, tetapi juga memahami alasan di balik setiap tindakan yang dilakukan.

---

# 🏢 Enterprise Mini Project

Mini Project merupakan tahap integrasi dari seluruh materi yang telah dipelajari.

Berbeda dengan Hands-on Lab maupun Challenge Lab, Mini Project berfokus pada penyelesaian sebuah pekerjaan utuh yang menghasilkan deliverable nyata.

Contoh implementasi Mini Project antara lain:

- Enterprise Server Health Audit Report.
- Linux Identity Management Implementation.
- Enterprise Backup System.
- Secure Web Server Deployment.
- Infrastructure Monitoring Report.
- Cloud Infrastructure Deployment.

Setiap Mini Project didokumentasikan layaknya laporan teknis yang dapat diserahkan kepada Senior Engineer sebagai hasil pekerjaan.

# 📝 Technical Documentation

Dokumentasi tidak hanya berisi hasil akhir, tetapi juga mencakup:
